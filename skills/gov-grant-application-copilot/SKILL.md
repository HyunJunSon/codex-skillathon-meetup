# 정부 지원사업 신청 코파일럿

## 개요

한국 스타트업 및 중소기업이 공공 지원사업 탐색부터 제출 후 추적까지 전체 신청 워크플로우를 수행할 수 있도록 지원합니다. 공공 공고, 회사 프로필 근거 자료, 설정 가능한 기업 유형 및 사업 유형 선호도, 사람 검토 게이트를 결합합니다.

이 스킬은 기본적으로 한국어 산출물을 지원하되, 기계 판독 가능한 아티팩트는 안정적인 영문 파일명을 유지합니다.

## 사용 시점

다음과 같은 요청이 있을 때 이 스킬을 사용합니다:

- 한국 정부 보조금, 지원사업, 바우처, 정책자금, R&D 과제, 수출 지원, 고용 보조금, 지자체 프로그램 찾기
- K-Startup, 기업마당, 중소벤처24, 지자체 또는 관련 기관 공고의 적격성 비교
- Google Drive 또는 로컬 문서에서 회사 정보를 추출하여 신청서 초안 준비
- 상위 지원사업 추천, 필요 서류 체크리스트, 일정, 검토 보고서, 후속 추적기 생성
- API 키가 없을 때 fixture 폴백을 활용한 Docker 또는 Cloudflare Tunnel 데모 구축

## 필수 입력

누락된 핵심 정보를 요청하되, 한 번에 세 가지 이상 질문하지 않습니다. 사용자가 빠른 진행을 원하면 보수적 가정을 세우고 라벨링합니다.

- 회사명
- 사업자등록번호 또는 법인등록번호 (표시 시 마스킹)
- 업종, 아이템 또는 제품 설명
- 설립일 또는 업력
- 회사 소재지
- 대표자 및 신청 담당자 연락처 (필요 시 마스킹)
- 기업 유형, 기본값 `초기창업기업`
- 사업 유형, 기본값 `바우처`, `정책자금`, `인력지원`

선택 입력:

- Google Drive 폴더 링크 또는 폴더 ID
- 로컬 폴더 폴백
- 기존 회사소개서, IR 덱, 재무제표, 4대보험 가입자 명부, 인증서, 특허, 이전 신청서
- 제외 조건, 선호 지역/기관, 준비 가능 기간, 내부 검토자, 마감일 버퍼
- K-Startup, 기업마당, 중소벤처24 등 공공 API 키

## 설정 가능 범위

기업 유형은 자연어, CLI 플래그, 웹 UI 다중 선택, 또는 `company.profile.json`의 `target_profile.company_stage`로 설정할 수 있습니다.

허용 기업 유형:

- `예비창업자`
- `초기창업기업`
- `3년 미만 창업기업`
- `7년 미만 창업기업`
- `중소기업`
- `소상공인`
- `사회적기업`
- `여성기업`
- `장애인기업`
- `벤처기업`
- `연구소기업`
- `수출기업`

사업 유형은 자연어, CLI 플래그, 웹 UI 다중 선택, 또는 `company.profile.json`의 `target_profile.program_types`로 설정할 수 있습니다.

허용 사업 유형:

- `바우처`
- `정책자금`
- `인력지원`
- `사업화`
- `R&D`
- `수출/글로벌`
- `판로/마케팅`
- `시설/장비`
- `창업교육/멘토링`
- `투자/보증`
- `지자체 지원`

CLI 예시:

```bash
python scripts/validate_gov_grant_config.py company.profile.json
python scripts/validate_gov_grant_output.py result.md
```

애플리케이션 구현체는 `--company-stage`, `--program-types`, 웹 UI 다중 선택 컨트롤, 프로필 JSON 필드로 동일한 설정을 노출해야 합니다.

## 멀티 에이전트 워크플로우

사용자가 요청한 산출물에 필요한 최소한의 에이전트 세트를 사용합니다. 전체 실행 시 아래 순서로 실행하며 각 에이전트의 근거 추적을 보존합니다.

1. 소스 수집 에이전트
   - K-Startup, 기업마당, 중소벤처24 및 설정된 공공 데이터 소스를 조회합니다.
   - 기관, 사업명, 사업 유형, 지원 대상, 마감일, 지원 내용, 신청 방법, 소스 URL, 첨부 URL, 수집 시각을 정규화합니다.
   - 사업명, 기관, 마감일, 소스 URL 기준으로 중복을 제거합니다.
   - `programs.normalized.json`을 출력합니다.

2. 회사 프로필 에이전트
   - Drive 폴더가 제공되면 `gcli`를 사용하여 Google Drive 인증, 폴더 목록, 다운로드, 내보내기를 수행합니다.
   - Drive를 사용할 수 없으면 로컬 폴더로 폴백합니다.
   - 소스 파일명, 페이지/섹션(가능한 경우), 신뢰도와 함께 회사 정보를 추출합니다.
   - `company.profile.json`과 `missing_company_fields.md`를 출력합니다.

3. 적격성 에이전트
   - 지원 대상, 제외 조건, 업력, 소재지, 업종, 매출, 고용, 인증, R&D, 수출, 사업 유형 요건을 회사 프로필과 비교합니다.
   - 각 프로그램을 `eligible`(적격), `ineligible`(부적격), `needs_confirmation`(확인 필요)으로 분류합니다.
   - `eligibility.matrix.md`와 `blocking_issues.md`를 출력합니다.

4. 랭킹 에이전트
   - 설정된 가중치로 적격 및 불확실 프로그램에 점수를 매깁니다.
   - 선택된 사업 유형과 기업 유형을 랭킹 우선순위로 취급하되, 자동 적격 판정으로 사용하지 않습니다.
   - 상위 3개 추천과 불확실 항목이 포함된 `priority.ranking.md`를 출력합니다.

5. 문서 작성 에이전트
   - 초안 작성 전 공고문과 필수 신청서 양식 구조를 읽습니다.
   - 출처가 있는 회사 데이터로 알려진 사실을 채우고, 생성된 검토 텍스트는 `generated`로 표시합니다.
   - 누락되거나 위험한 필드는 `확인 필요`로 표시합니다.
   - `application.draft.md`, `business_plan.sections.md`, `fields_to_confirm.md`를 출력합니다.

6. 일정 에이전트
   - 마감일로부터 역산하여 증명서 발급, 초안 작성, 검토, 서명, 제출에 대한 버퍼를 추가합니다.
   - 사용자가 명시적으로 수락하지 않는 한 내부 마일스톤의 주말 마감을 피합니다.
   - `submission.timeline.md`와 `calendar_events.csv`를 출력합니다.

7. 검토 및 컴플라이언스 에이전트
   - 누락 서류, 숫자 불일치, 이름/주소/대표자 불일치, 미충족 요건, 마감 리스크, 개인정보 노출, 전문가 검토 필요 사항을 확인합니다.
   - `pre_submission_review.md`와 `risk_register.md`를 출력합니다.

8. 제출 후 추적 에이전트
   - 접수 확인, 보완 요청, 서류 심사, 발표 심사, 협약, 집행, 정산 증빙을 추적합니다.
   - `post_submission_tracker.md`를 출력합니다.

## 공공 API 연동

자격 증명과 승인이 가능한 경우 실제 공공 API 연동을 우선합니다. API 키, 승인, 서버 IP 등록, 네트워크 접근, 속도 제한이 차단하는 경우에만 fixture 폴백을 사용합니다.

주요 소스:

- K-Startup 공공데이터포털 API: `창업진흥원_K-Startup(사업소개,사업공고,콘텐츠 등)_조회서비스`
- 기업마당 지원사업 정보
- 중소벤처24 공고 및 신청 정보

보조 소스:

- 정부24 및 지자체 공고
- 관련 시 조달/정부 구매 공고
- 명시적으로 필요하고 승인된 경우의 사업자 상태, 고용, 특허, 인증, 벤처 확인 API

연동 규칙:

- API 키는 환경변수 또는 시크릿 매니저에만 저장합니다.
- 모든 소스를 적격성 또는 랭킹 전에 `programs.normalized.json`으로 정규화합니다.
- 가능한 경우 `source_url`, `source_name`, `fetched_at`, `raw_id`를 포함합니다.
- 사용자가 디버깅을 요청하지 않는 한 원시 API 응답을 최종 사용자 대면 요약에 포함하지 않습니다.
- API가 서버 IP 등록을 요구하는 경우, 로컬 Docker와 Cloudflare Tunnel이 충분한지 명시합니다. Cloudflare Tunnel은 고정 아웃바운드 IP를 제공하지 않습니다.

## gcli를 통한 Google Drive 연동

사용자가 폴더 접근을 제공하거나 확인한 경우에만 Google Drive를 사용합니다. 구현체는 인증 및 파일 작업에 `gcli` 명령을 사용해야 합니다.

권장 흐름:

1. Drive 폴더 ID 또는 링크와 의도된 문서 범위를 확인합니다.
2. 아직 인증되지 않았으면 `gcli` 인증을 실행합니다.
3. 선택된 폴더의 파일을 나열합니다.
4. 지원되는 파일을 작업 디렉토리로 다운로드 또는 내보냅니다.
5. 가능한 경우 로컬에서 텍스트를 추출합니다.
6. `company.profile.json`에 소스 메타데이터를 기록합니다.

Drive 안전 규칙:

- 사용자의 전체 Drive를 크롤링하지 않습니다.
- 명시적 지시 없이 민감한 회사 문서를 외부 서비스에 업로드하지 않습니다.
- 데모에는 샘플 또는 더미 Drive 폴더를 선호합니다.
- 필드에 필요하고 승인된 경우가 아니면 요약에서 주민등록번호, 사업자등록번호, 전화번호, 계좌번호, 개인 이메일을 마스킹합니다.

## Docker 및 Cloudflare 배포

실행 가능한 데모를 구현할 때 로컬 Docker Compose 및 Cloudflare Tunnel과 호환되는 배포 지침을 유지합니다.

예상 배포 동작:

- 앱과 워커가 Docker Compose에서 로컬 실행됩니다.
- API 키, Drive 자격 증명, 기본 인증 자격 증명, 터널 토큰은 `.env` 또는 호스트 시크릿 매니저를 통해 주입됩니다.
- 공공 API 자격 증명 없이도 데모용 fixture 데이터를 사용할 수 있습니다.
- 헬스 엔드포인트가 앱, 워커, fixture 로더, 설정된 API 클라이언트를 확인합니다.
- Cloudflare Tunnel은 데모 인증이 활성화된 후에만 로컬 서비스를 노출합니다.

배포 주의사항:

- `.env`, Drive 자격 증명, API 키, 생성된 토큰, 원시 민감 문서 내보내기를 커밋하지 않습니다.
- Cloudflare Tunnel은 인바운드 접근을 위한 외부 URL을 제공하며, 고정 아웃바운드 IP가 필요한 공공 API 문제를 해결하지 않습니다.
- 공개 데모 URL은 원시 회사 문서나 마스킹되지 않은 개인 데이터를 노출해서는 안 됩니다.

## 안전 규칙

- 신청서 초안을 검토용 초안으로 취급하며, 최종 제출물이 아닙니다.
- 사실, 가정, 생성된 텍스트, 누락 필드를 분리합니다.
- 적격성 및 회사 정보에 대해 소스 파일명 또는 공공 URL을 인용합니다.
- 모호한 요건은 `확인 필요`로 표시합니다.
- 세무, 법률, 회계, 노무, 개인정보, 협약 해석은 전문가 검토로 플래그합니다.
- 인증서, 특허, 매출, 인원, 투자, 수상, 파트너십, 이전 선정 이력을 절대 날조하지 않습니다.
- 적격성을 과대 표현하지 않습니다. 불확실한 유망 매칭은 `준비` 또는 `사람 판단 필요`이며, `지금 신청`이 아닙니다.
- 출력물에서 민감 데이터를 최소화합니다. 신청 필드에 필요한 경우가 아니면 식별자를 마스킹합니다.
- 제출 전 사람 승인 게이트를 보존합니다.

## 우선순위 로직

기본 점수 가중치:

- 적격성 적합도: 35%
- 전략적 관련성: 20%
- 지원 규모 또는 혜택 가치: 15%
- 마감일 실현 가능성: 15%
- 준비 노력: 10%
- 과거 성공 가능성 또는 근거 강도: 5%

결정 라벨:

- `지금 신청`: 높은 적합도와 실현 가능한 마감일/서류 준비 상태
- `준비`: 높은 적합도이나 서류 또는 요건 확인 필요
- `모니터링`: 미개시, 향후 차수 예상, 또는 현재 낮은 긴급도
- `신청 불가`: 핵심 요건 미충족
- `사람 판단 필요`: 모호한 해석 또는 전문가 검토 필요

```markdown
## 데이터 소스

```

품질 요건:

- 충분한 프로그램이 있을 때 상위 3개 추천을 포함합니다.
- 각 추천 프로그램에 대한 근거와 불확실 항목을 포함합니다.
- 제출 전 최소 하나의 명시적 사람 검토 체크포인트를 포함합니다.
- 회사 정보와 공공 공고 정보에 대한 소스 라벨을 포함합니다.

## 리소스

상세한 에이전트 책임, 아티팩트 스키마, gcli 흐름, API 정규화, 배포 가이드, 검토 체크리스트는 `references/playbook.md`를 참조합니다.

생성된 마크다운을 출력 계약에 대해 검증하려면 `scripts/validate_gov_grant_output.py`를 사용합니다.

`company.profile.json` 대상 프로필과 필수 회사 필드를 검증하려면 `scripts/validate_gov_grant_config.py`를 사용합니다.
